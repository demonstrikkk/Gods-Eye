## 2. Agentic Workflow Pipeline

### The 10-Step Agent Execution Flow

```
USER QUERY    │    ▼[1] QUERY PARSING & INTENT DETECTION    ├─ Extract entities (people, places, concepts)    ├─ Detect domain(s): Economic | Geopolitical | Social | Climate | Policy | Risk | Simulation    ├─ Classify request type: Analysis | Prediction | Simulation | Comparison    └─ Enrich with context (recent signals, cached data)    │    ▼[2] STRATEGIC AGENT INITIALIZATION    ├─ Select relevant expert agents based on domain(s)    ├─ Load agent configurations & model preferences    ├─ Prepare knowledge graph access    └─ Initialize tool access (OSINT, maps, diagrams)    │    ▼[3] PARALLEL EXPERT AGENT EXECUTION    ├─ AgentA (Economic)      ────┐    ├─ AgentB (Geopolitical)  ────┼─ asyncio.gather() - ALL IN PARALLEL    ├─ AgentC (Climate)       ────┤    ├─ AgentD (Social)        ────┤    ├─ AgentE (Policy)        ────┤    └─ AgentF (Risk)          ────┘    │    │ Each Agent Workflow:    │ ┌─────────────────────────────────────────┐    │ │ A. Fetch relevant data/context          │    │ │ B. Generate initial assessment          │    │ │ C. Structure as AgentClaim              │    │ │ D. Compute confidence_score (0.0-1.0)   │    │ │ E. Trace evidence/source citations      │    │ │ F. Return structured JSON with:         │    │ │    - claim: str                         │    │ │    - confidence_score: float            │    │ │    - reasoning: str                     │    │ │    - evidence: [sources]                │    │ │    - methodology: str                   │    │ └─────────────────────────────────────────┘    │    ▼[4] CONFIDENCE DIVERGENCE CHECK    │    ├─ Calculate: max_confidence - min_confidence    │    ├─ If divergence ≤ 0.25 ──────────────────┐    │                                          │    │  ▼ (SKIP DEBATE)                        ▼ If divergence > 0.25 & ≥3 agents    │  [GO TO CONSENSUS]                  [5] TRIGGER AGENT DEBATE SYSTEM    │                                         │    │                                         ▼    │                                     DEBATE MECHANISM    │                                     ┌─────────────────┐    │                                     │ PHASE 1: OPENING│    │                                     │ Each agent states    │                                     │ their position    │                                     └────────┬────────┘    │                                              │    │                                     ┌────────▼────────┐    │                                     │ PHASE 2: EVIDENCE    │                                     │ Present data,    │                                     │ cite sources    │                                     └────────┬────────┘    │                                              │    │                                     ┌────────▼────────────────┐    │                                     │ PHASE 3: CROSS-EXAM     │    │                                     │ Agents question each    │    │                                     │ other's logic & data    │    │                                     └────────┬────────────────┘    │                                              │    │                                     ┌────────▼────────┐    │                                     │ PHASE 4: REBUTTAL    │                                     │ Counter-arguments    │                                     │ defend positions    │                                     └────────┬────────┘    │                                              │    │                                     ┌────────▼────────┐    │                                     │ PHASE 5: CLOSING│    │                                     │ Final statements    │                                     │ summary positions    │                                     └────────┬────────┘    │                                              │    │                                     ┌────────▼──────────────┐    │                                     │ PHASE 6: DELIBERATION │    │                                     │ Evaluator LLM votes   │    │                                     │ weighs arguments      │    │                                     └────────┬──────────────┘    │                                              │    └──────────────────────────────┬──────────────┘  [Debate can loop max 3x]                                   │                                   ▼[6] CONSENSUS BUILDING    │    ├─ Aggregate all agent claims (post-debate)    ├─ Apply voting mechanisms:    │  ├─ Majority voting (simple)    │  ├─ Confidence-weighted voting    │  ├─ Domain-weighted voting    │  ├─ Novelty/Recency weighting    │  ├─ Expert consensus (Delphi method)    │  └─ Bayesian aggregation    │    ├─ Generate output components:    │  ├─ Executive Summary (1-3 sentences)    │  ├─ Detailed Assessment    │  ├─ Minority Opinions (if any divergence survived)    │  ├─ Causal Chains (how A leads to B leads to C)    │  ├─ Uncertainty Analysis (confidence intervals)    │  └─ Risk Assessment & Recommendations    │    └─ Finalize structured OrchestratedResponse    │    ▼[7] TOOL & OSINT EXECUTION (Parallel)    │    ├─ OSINT Fetchers ─────────────────────┐    │  ├─ GDELT (global events)            │    │  ├─ FRED/World Bank (economics)      │    │  ├─ USGS (earthquakes)               │    │  ├─ NASA EONET (wildfires/disasters) │    │  ├─ OpenSky (aviation)               │    │  ├─ Open-Meteo (weather)             │    │  ├─ CISA KEV (exploits)              │    │  └─ Social feed (Reddit/Mastodon)    │ ALL PARALLEL    │                                      │    ├─ Visual Generators ───────────────────┤    │  ├─ Chart builder (matplotlib/plotly)│    │  ├─ Diagram generator (flowcharts)  │    │  └─ Intent visualizer               │    │                                      │    ├─ Map Intelligence ───────────────────┤    │  ├─ Geospatial query processor       │    │  ├─ Constituency/ward lookup         │    │  └─ Infrastructure mapper            │    │                                      │    └─ Live Signal Fetcher ────────────────┘       └─ From runtime_state.json    │    ▼[8] RESPONSE ASSEMBLY    │    ├─ Combine reasoning + visuals + maps + tools    ├─ Format for frontend consumption    ├─ Tag data origin: REAL | MOCKED | SYNTHESIZED    └─ Add timestamps & metadata    │    ▼[9] GRACEFUL DEGRADATION CHECK    │    ├─ If any service failed:    │  ├─ Return what succeeded    │  ├─ Mark failures in response    │  ├─ Use fallback/cached data    │  └─ NO COMPLETE FAILURE    │    └─ Persist state to runtime_state.json    │    ▼[10] RESPONSE DELIVERY    │    └─ Return structured JSON to frontend with:       ├─ Executive briefing       ├─ Detailed report       ├─ Visual assets (charts/diagrams)       ├─ Map data (if geographic)       ├─ Confidence/uncertainty metrics       ├─ Source citations       └─ Data freshness timestamps
```

## Summary: Gods-Eye OS as a Reactive Intelligence Platform

The God's Eye OS is fundamentally a **reactive, multi-agent command and control platform**:

1.  **Input:** Natural language queries from strategic/civic decision-makers
2.  **Processing:**
    -   Automatic intent/domain detection
    -   Parallel expert agent reasoning (Groq LLMs)
    -   Forced debate if agents disagree
    -   Consensus synthesis
    -   Tool execution (visuals, maps, live data)
3.  **Output:** Structured intelligence briefings with:
    -   Executive summaries
    -   Confidence metrics with uncertainty bounds
    -   Causal reasoning chains
    -   Visual dashboards (charts, maps, data flows)
    -   Source citations & data freshness
    -   Risk assessments & recommendations
4.  **Resilience:** Graceful degradation - if any service fails, remaining services complete and return partial results
5.  **Speed:** 4-7 second end-to-end latency via aggressive parallelism

The system blends **real live data feeds** (13+ APIs for global OSINT) with **seeded civic context** (Indian constituencies, schemes, workers) to provide a unified world model suitable for both macro-strategic decisions and micro-governance operations.
