4) End-to-End Workflow (How It Actually Runs)
Startup flow:
1. Backend starts.
2. Feed engine begins periodic feed pulls.
3. Runtime engine begins periodic source pulls + synthesis.
4. Runtime state updates are persisted.
5. Frontend polls data endpoints and updates UI.
Global intelligence flow:
1. Runtime engine fetches multiple external sources.
2. Converts source outputs into normalized signal objects.
3. Resolves/attaches country and layer context.
4. Builds enriched countries + alerts + source health.
5. Exposes via global endpoints and map/graph payloads.
Feed intelligence flow:
1. RSS/news pulled concurrently.
2. Deduped and categorized.
3. Stored as feed briefs.
4. Optionally forwarded to text-intelligence processing pipeline.
Strategic analysis flow:
1. User query posted to intelligence endpoint.
2. Planner selects tools/data paths.
3. Tools execute with timeouts/fallbacks.
4. LLM synthesis generates structured strategic output.
5. If sources fail, fallback output is explicit (not silent hallucination).
Civic ingestion flow:
1. Unstructured text arrives at ingestion endpoint.
2. Background processing extracts entities/sentiment/action suggestions.
3. Graph linkage write path available through Neo4j integration.
Frontend user flow:
1. Command center loads.
2. Status/ticker/markets/nav bars render.
3. Main map/globe and intelligence sidebar coordinate selected context.
4. Pages/tabs query dedicated backend endpoints for drilldown.
